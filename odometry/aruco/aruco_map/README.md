# aruco_map

ROS 2 пакет для публикации карты ArUco-маркеров из файла.
Карта публикуется как latched (`transient_local`) топик и используется нодами детекции и локализации.

---

## Назначение

Нода `aruco_map`:
- читает описание ArUco-маркеров из текстового файла
- публикует массив маркеров (`MarkerArray`) в мировом frame
- периодически перепубликует карту, чтобы новые подписчики сразу её получили

Типичное применение — источник `/map_markers` для `aruco_loc_node`.

---

### aruco_map

**Публикации:**
- `map_markers` — `aruco_det_loc/msg/MarkerArray`  
  (QoS: `transient_local`, `reliable`)

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|--------|----|-------------|---------|
| `frame_id` | string | `world` | Мировая система координат карты |
| `marker_size` | double | `0.16` | Размер маркеров по умолчанию (м) |
| `topic_name` | string | `map_markers` | Имя публикуемого топика |
| `map_file` | string | `config/markers.txt` | Файл с описанием маркеров |

---

## Формат файла карты

Текстовый файл, по одному маркеру на строку:

```text
# id  size   x    y    z   yaw  pitch  roll
0   0.16   1.0  0.0  0.5   0    0     1.57
1   0.16   2.0  1.0  0.5   0    0     0.0
```

---

## Пример запуска

```bash
ros2 run aruco_map aruco_map_node
```

После запуска карта будет доступна в топике:

```bash
/map_markers
```

---


Без этого топика `aruco_loc_node` не сможет вычислить позу.

## Генерация файла карты (genmap.py)

Файл карты может быть сгенерирован с помощью утилиты `genmap.py` из пакета `aruco_pose`.

Базовый формат команды:

```bash
ros2 run aruco_pose genmap.py LENGTH X Y DIST_X DIST_Y FIRST_ID [--bottom-left] [-o MAP_NAME]
```

Где:

- `LENGTH` — размер маркера (в метрах)
- `X` — количество маркеров по оси X
- `Y` — количество маркеров по оси Y
- `DIST_X` — расстояние между центрами маркеров по оси X (в метрах)
- `DIST_Y` — расстояние между центрами маркеров по оси Y (в метрах)
- `FIRST_ID` — ID первого (левого нижнего) маркера
- `MAP_NAME` — имя файла карты (например, `new_map.txt`).  
  Если указано, файл будет сохранён в директорию пакета `aruco_map`: `<share/aruco_map>/config/` (путь определяется через `ament_index_python`).

Дополнительно, утилита старается сделать **дубликат** файла в исходниках репозитория:
`sverk_ws/src/sverk_drone/odometry/aruco/aruco_map/config/` (если этот workspace найден на диске).
- `--bottom-left` (опционально) — начинать нумерацию маркеров с левого нижнего угла

Пример:

```bash
ros2 run aruco_pose genmap.py 0.33 2 4 1 1 0 --bottom-left -o new_map.txt
```

Дополнительную информацию по утилите можно получить по ключу `-h`:

```bash
ros2 run aruco_pose genmap.py -h
```

## Опциональное визуальное отображение выбранной маркерной карты

Нода поддерживает аргументы "publish_debug_images" и "debug_draw_origin", которые по умолчанию имеют значения "true".

- "publish_debug_images" - включение/выключение публикации рендеров используемых карт.
- "debug_draw_origin" - включение/выключение отображения мирового начала координат на рендере (отображение происходит ТОЛЬКО в том случае, если плоскость (см. далее) включает начало координат)

При включении этих аргументов, нода выполняет:
- Поиск всех плоскостей на карте (маркеры на карте могут иметь произвольное расположение; группа маркеров, находящихся на одной плоскости называется плоскостью)
- Каждая плоскость публикуется в отдельный топик с названиями: "/aruco_map/debug_image/plane_1", ".../plane_2", ...
- Если debug_draw_origin = true и начало координат пересекает плоскость, вы увидите графический глиф, отображающий оси праворукой системы координат на плоскости.

## Другие параметры ноды aruco_map

Нода содержит дополнительные функции вроде добавления подписей с id-номерами маркеров на рендер для дебага, изменения параметров отображения глифа начала координат и др. Ознакомиться с ними можно в скрипте ноды, в конструкторе класса.

## Сервис переключения/перезагрузки карты

Нода поддерживает сервис, позволяющий в runtime переключить файл карты или перечитать уже активный файл (например, после редактирования на диске), без перезапуска `aruco_map_node`.

### `/aruco_map/set_map_file`

**Тип:** `aruco_map/srv/SetMapFile`

**Request:**
- `string map_file` — путь к файлу карты (абсолютный или относительный)

**Response:**
- `bool success` — успешно ли загружено/переключено
- `string message` — описание ошибки или подтверждение
- `string active_map_file` — итоговый активный файл (абсолютный путь)
- `uint32 marker_count` — количество маркеров в активной карте
- `uint32 plane_count` — количество плоскостей (для debug render’ов)

### Поддерживаемые сценарии использования

- **Переключение или перезапуск по имени файла**  
  `map_file: "markers2.txt"`  
  Относительные пути решаются относительно директории текущего активного файла (обычно `.../aruco_map/config/`).

- **Переключение по относительному пути с подпапками**  
  `map_file: "experiments/markers2.txt"`  
  Допускаются подпапки.

- **Переключение по абсолютному пути**  
  `map_file: "/home/sverk/.../aruco_map/config/markers2.txt"`  

### Правила валидации

- Пробелы по краям строки игнорируются.
- Пустая строка запрещена.
- Пути, содержащие `..` (parent traversal), запрещены.
- При ошибке открытия/парсинга нода **не** переключается и продолжает публиковать предыдущую (последнюю корректную) карту.

### Примеры вызова

```
ros2 service call /aruco_map/set_map_file aruco_map/srv/SetMapFile "{map_file: 'markers2.txt'}"
```
```
ros2 service call /aruco_map/set_map_file aruco_map/srv/SetMapFile "{map_file: '/home/sverk/.../aruco_map/config/markers2.txt'}"
```
